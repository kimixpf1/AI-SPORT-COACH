# 测试指南

## 🧪 测试真实杠铃检测功能

### 准备工作
1. 启动开发服务器：
   ```bash
   npm run dev
   ```
2. 访问：http://localhost:3000
3. 准备一个举重视频（建议特征见下方）

### 理想测试视频特征

#### ✅ 推荐特征
- **动作类型**：深蹲、硬拉、卧推等标准动作
- **拍摄角度**：侧面 90 度
- **光线**：明亮、均匀
- **背景**：简洁、单色
- **杠铃片**：清晰可见、圆形完整
- **分辨率**：720p 或更高
- **时长**：5-15 秒
- **帧率**：30fps 或更高

#### ❌ 避免特征
- 逆光拍摄
- 背景复杂（多人、多器械）
- 杠铃片被遮挡
- 视频模糊或抖动
- 杠铃片太小（远距离拍摄）

### 预期结果

#### 成功检测
- 检测到 20-30 个轨迹点（取决于视频时长）
- 轨迹连续、平滑
- 速度和加速度曲线合理
- 控制台无错误信息

#### 检测失败
如果检测失败，控制台会显示：
```
帧 X/30: 未检测到杠铃
```

如果大部分帧都未检测到，会弹出提示：
```
检测失败：未能检测到足够的杠铃位置
```

### 调试技巧

#### 1. 查看控制台日志
打开浏览器开发者工具（F12），查看 Console 标签：
- OpenCV 加载状态
- 每帧检测结果
- 检测到的坐标
- 错误信息

#### 2. 检查检测参数
如果检测效果不佳，可以尝试调整 `VideoTracker.tsx` 中的参数：

```typescript
cv.HoughCircles(
  blurred,
  circles,
  cv.HOUGH_GRADIENT,
  1,                    // dp: 累加器分辨率
  blurred.rows / 8,     // minDist: 圆心最小距离
  100,                  // param1: Canny边缘检测高阈值 ⬅️ 可调整
  30,                   // param2: 累加器阈值 ⬅️ 可调整（越小检测越多）
  15,                   // minRadius: 最小半径 ⬅️ 可调整
  150                   // maxRadius: 最大半径 ⬅️ 可调整
);
```

**调整建议**：
- `param2` 降低（如 20）：检测更多圆，但可能误检
- `param2` 提高（如 40）：检测更严格，但可能漏检
- `minRadius/maxRadius`：根据视频中杠铃片的实际大小调整

#### 3. 测试不同视频
尝试不同特征的视频，找出最佳拍摄条件。

### 性能测试

#### 预期性能
- **OpenCV 加载**：2-5 秒（首次）
- **单帧检测**：50-200ms
- **总检测时间**：10-30 秒（30 帧）
- **内存占用**：< 200MB

#### 性能问题排查
如果检测很慢：
1. 检查视频分辨率（过高会影响性能）
2. 检查浏览器性能（关闭其他标签页）
3. 查看 CPU 使用率

### 已知问题

1. **首次加载慢**：OpenCV.js 约 8MB，首次加载需要时间
2. **检测不稳定**：某些帧可能检测失败，这是正常的
3. **轨迹跳跃**：如果杠铃片被遮挡，轨迹可能不连续
