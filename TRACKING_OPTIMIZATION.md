# 追踪算法优化说明

## 🎯 优化内容

### 1. 更严格的检测参数
**之前**：检测到 33 个圆（太多误检）
```typescript
param1: 80,   // Canny 阈值
param2: 20,   // 累加器阈值
minRadius: 10,
maxRadius: 200
```

**现在**：减少误检
```typescript
param1: 100,  // 提高 Canny 阈值
param2: 35,   // 提高累加器阈值
minRadius: 30, // 杠铃片通常较大
maxRadius: 120
```

### 2. 智能选择策略

**之前**：简单选择半径最大的圆
- 问题：可能选到灯、空调等其他圆形物体

**现在**：基于多因素评分系统
```typescript
得分 = 半径得分 + 位置得分 + 连续性得分
```

#### 评分因素：

**A. 半径得分（50分）**
- 40-100 像素：50分（杠铃片的典型大小）
- 30-120 像素：30分
- 其他：10分

**B. 位置得分（30分）**
- y > height/3：30分（杠铃通常在画面中下部）
- 其他：0分

**C. 连续性得分（最高50分）**
- 距离前一帧位置越近，得分越高
- 公式：`50 - distance/4`
- 最大移动距离：200像素

### 3. 轨迹连续性

**之前**：每帧独立检测，不考虑前后关系

**现在**：
- 传递前一帧的位置给检测函数
- 优先选择靠近前一帧位置的圆
- 减少轨迹跳跃

## 📊 预期效果

### 检测数量

## 🔧 如果效果仍不理想

### 调整半径范围
如果杠铃片在视频中很小或很大：
```typescript
minRadius: 20,  // 降低最小半径
maxRadius: 150  // 提高最大半径
```

### 调整位置权重
如果杠铃在画面上部：
```typescript
if (y > tempCanvas.height / 4) {  // 改为 1/4
  score += 30;
}
```

### 调整连续性权重
如果轨迹仍然跳跃：
```typescript
if (distance < 200) {
  score += Math.max(0, 100 - distance / 2);  // 提高连续性权重
}
```

### 调整检测阈值
如果检测到的圆太少：
```typescript
param2: 30,  // 降低阈值
```

如果检测到的圆太多：
```typescript
param2: 40,  // 提高阈值
```

## 📈 评分系统详解

### 示例 1：理想的杠铃片
```
位置: (450, 700)  // 画面中下部
半径: 60 像素
距离前一帧: 20 像素

得分计算：
- 半径得分: 50 (40-100 范围内)
- 位置得分: 30 (y > height/3)
- 连续性得分: 45 (50 - 20/4)
总分: 125
```

### 示例 2：天花板上的灯
```
位置: (300, 100)  // 画面上部

- 半径得分: 30 (30-120 范围内)
- 位置得分: 0 (y < height/3)
- 连续性得分: 0 (距离太远)
总分: 30
```

## 🎯 优化目标

1. **准确性**：选择正确的杠铃片
2. **连续性**：轨迹平滑不跳跃
3. **鲁棒性**：处理遮挡和光线变化

## 💡 未来改进方向

1. **卡尔曼滤波**：进一步平滑轨迹
2. **颜色过滤**：利用杠铃片的颜色特征
3. **形状匹配**：使用模板匹配
4. **深度学习**：使用 YOLO 或其他目标检测模型
