# 举重教练AI视频分析系统 - 技术方案报告

## 一、项目概述

**项目名称**：举重教练AI视频分析系统
**技术栈**：React 19 + Next.js 15 + TypeScript + Claude Opus 4.6 + OpenCV.js
**核心功能**：通过AI视觉分析举重训练视频，提供专业的技术指导和改进建议

## 二、视频处理技术方案

### 2.1 用户上传视频后的处理流程

**第一步：浏览器端杠铃追踪（客户端处理）**

1. 用户在浏览器中上传视频（支持MP4/MOV/AVI格式，最大50MB）
2. 视频在浏览器中加载，使用 **OpenCV.js** 进行实时杠铃追踪
3. 追踪过程：
   - 采样30帧（均匀分布在视频时长内）
   - 每帧使用Hough圆检测算法识别杠铃片位置
   - 记录每个时间点的杠铃坐标 (x, y, timestamp)
   - 计算相邻帧之间的速度和加速度
4. 追踪结果在浏览器中可视化展示（绿色轨迹线 + 红色当前位置点）

**第二步：服务器端视频帧提取（服务器处理）**

1. 视频文件通过HTTP POST上传到Next.js API路由 (`/api/analyze`)
2. 服务器接收视频Buffer，使用 **FFmpeg** 进行处理：
   - 提取8张关键帧（时间点分布在10%-90%位置，避免开头结尾）
   - 使用 **Sharp** 库压缩图片：
     - 分辨率：1280x720（保持宽高比）
     - 格式：JPEG
     - 质量：85%
   - 转换为Base64编码
3. 同时提取视频元数据：
   - 时长（duration）
   - 分辨率（width x height）
   - 帧率（fps）

**技术细节**：
- FFmpeg使用 `fluent-ffmpeg` 库封装
- 跨平台支持：macOS (ARM64/x64)、Linux、Windows
- 临时文件管理：使用系统临时目录，处理完成后自动清理

### 2.2 杠铃追踪算法详解

**使用的开源库**：
- **库名**：`@techstark/opencv-js` (版本 4.9.0)
- **加载方式**：从jsDelivr CDN动态加载（客户端）
- **CDN地址**：`https://cdn.jsdelivr.net/npm/@techstark/opencv-js@4.9.0-release.1/dist/opencv.js`

**核心算法**：Hough圆检测（HoughCircles）

**算法流程**：
```
1. 图像预处理
   ├─ 从视频帧提取ImageData
   ├─ 转换为OpenCV Mat格式
   ├─ 灰度化（cvtColor: RGBA → GRAY）
   └─ 高斯模糊（GaussianBlur, 9x9核, sigma=2）

2. Hough圆检测
   ├─ 方法：HOUGH_GRADIENT
   ├─ 参数：
   │  ├─ dp = 1（累加器分辨率）
   │  ├─ minDist = rows/8（圆心最小距离）
   │  ├─ param1 = 100（Canny边缘检测高阈值）
   │  ├─ param2 = 35（累加器阈值）
   │  ├─ minRadius = 30px（最小半径）
   │  └─ maxRadius = 120px（最大半径）
   └─ 输出：检测到的所有圆形对象

3. 智能选择策略（多候选圆评分）
   ├─ 半径得分（40-100px为最佳，权重50分）
   ├─ 位置得分（画面下半部分优先，权重30分）
   ├─ 连续性得分（与前一帧距离<200px，权重最高50分）
   └─ 选择得分最高的圆作为杠铃位置
```

**关键代码位置**：`app/components/VideoTracker.tsx:196-320`

### 2.3 速度和加速度计算

**计算公式**：

```typescript
// 速度计算（像素/秒）
const dt = timestamp[i] - timestamp[i-1];  // 时间差
const dx = x[i] - x[i-1];                  // x方向位移
const dy = y[i] - y[i-1];                  // y方向位移
const velocity = sqrt(dx² + dy²) / dt;     // 欧几里得距离 / 时间

// 加速度计算（像素/秒²）
const dv = velocity[i] - velocity[i-1];    // 速度差
const acceleration = dv / dt;              // 速度变化率
```

**数据结构**：
```typescript
interface TrackingData {
  trajectory: Array<{
    x: number;        // 像素坐标
    y: number;
    timestamp: number; // 秒
  }>;
  velocityData: Array<{
    time: number;
    velocity: number;      // px/s
    acceleration: number;  // px/s²
  }>;
}
```

**关键代码位置**：`app/components/VideoTracker.tsx:390-416`

## 三、AI分析技术方案

### 3.1 输入数据格式

**Claude API接收的输入**：

1. **8张JPEG图片**（Base64编码）
   - 格式：`image/jpeg`
   - 分辨率：最大1280x720
   - 编码：Base64字符串
   - 时间点：均匀分布在视频10%-90%位置

2. **视频元数据**（文本）
   - 格式：MP4/MOV/AVI
   - 时长：X.XX秒
   - 分辨率：宽x高
   - 帧率：XX.X fps
   - 关键帧时间点列表

3. **分析指令**（文本）
   - 要求识别动作类型
   - 要求分析杠铃轨迹
   - 要求评估速度和姿态
   - 要求返回JSON格式结果

**API调用示例**：
```typescript
const response = await client.messages.create({
  model: 'claude-opus-4-6',
  max_tokens: 4096,
  system: SYSTEM_PROMPTS.videoAnalysis,  // System prompt
  messages: [{
    role: 'user',
    content: [
      { type: 'image', source: { type: 'base64', media_type: 'image/jpeg', data: frame1 } },
      { type: 'image', source: { type: 'base64', media_type: 'image/jpeg', data: frame2 } },
      // ... 共8张图片
      { type: 'text', text: '作为专业举重教练，请分析这个视频...' }
    ]
  }]
});
```

**关键代码位置**：`lib/claude.ts:36-130`

### 3.3 调用方式

**一次性调用**：
- 所有8张图片 + 元数据 + 指令在一次API请求中发送
- 模型：Claude Opus 4.6
- Max tokens：4096
- 响应时间：约5-15秒（取决于网络和API负载）

**成本估算**：
- 输入：8张图片（约1.5MB压缩后）+ 文本提示词
- 输出：约1000-2000 tokens的JSON
- 单次分析成本：约 $0.05-0.10 USD

**错误处理**：
- JSON解析失败时使用正则提取关键信息
- 提供备用结构确保系统稳定性
- 保存原始响应用于调试

```prisma
model Analysis {
  id              String   @id @default(cuid())
  createdAt       DateTime @default(now())

  videoDuration   Float?                    // 视频时长

  overallScore    Float?                    // 总体评分

  @@index([createdAt])
}
```

**存储内容**：
- 分析结果的所有字段

**数据访问**：
- 使用Prisma Client进行类型安全的数据库操作
