# 项目框架

## 当前定位

- 项目名称：AI 运动教练
- 当前核心链路：上传训练视频 → MediaPipe 本地识别 → 轨迹与速度分析 → 输出动作建议 → GitHub Pages 部署
- 当前优先级：先保证线上 Pages 可访问，再继续细化动作识别和移动端体验

## 当前目录结构

- `app/`：Next.js App Router 页面与组件
- `app/components/`：视频轨迹、速度图表等核心前端组件
- `lib/`：动作分析类型与 MediaPipe 本地分析逻辑
- `.github/workflows/`：GitHub Pages 自动部署流程
- 根目录文档：记录实现状态、优化方案、测试说明与项目长期记录

## 当前核心实现

- 已接入 MediaPipe 本地姿态识别
- 已支持深蹲、卧推、高翻、硬拉与自动识别
- 已输出轨迹、速度、加速度、评分、风险与建议
- 已补充关键姿态指标与动作时间线复盘面板
- 已改为静态导出并通过 GitHub Actions 发布到 `gh-pages` 分支
- 当前进入识别鲁棒性增强阶段，重点提升多角度、多分辨率、低清视频下的兼容能力
- 已加入机位识别、清晰度/覆盖度评估、低质视频降级分析与更专业的训练建议输出
- 已进一步补充侧后方、后方机位识别与后链/下肢对线分析能力
- 已开始针对手机端小屏和微信场景优化上传与结果阅读体验

## 当前主要问题

- GitHub Pages 地址已经可以打开，但当前真实测试视频仍可能提示“无法分析，请更换更清晰的视频”
- 当前已定位到线上真实视频失败还包含 MediaPipe 运行时资源通过 CDN 加载时的 MIME 异常，正在补充更稳定的运行时候选源
- 电脑端界面整体布局与专业感不足，左右区域、按钮层级与信息密度仍需优化
- 当前需要先提升真实视频识别成功率与专业反馈质量，再继续把更完整的后方机位分析增强和手机端优化稳定同步到线上
- 当前本地代码已完成识别兜底与界面重构，但远端推送仍受当前环境下的 Git / GitHub CLI 认证异常影响，线上尚未同步本轮修复
- 本轮当前目标已切换为：优先把本地修复同步到远端，并直接使用真实平台网址完成上传与分析复测
- 已通过远端文件更新方式把 Pages 运行时热修复推到线上，直接用真实平台网址复测后确认：WASM 资源已能正常请求，但分析流程仍卡在 MediaPipe 初始化阶段，正在继续热修复
- 用户最新桌面端报错已明确指向 MediaPipe `ImageToTensorCalculator` 的 ROI 宽高为 0，需要从视频抽帧、seek 边界、元素尺寸与异常兜底四个层面一起修
- 用户最新微信端报错为分析完成后的 client-side exception，说明移动端除了识别链路还需要补结果渲染与异常恢复
- 当前电脑端布局仍不符合目标，需要参考原始复刻项目调整为更规整的并排模块，减少上右侧无效大卡片并修正按钮文字断行
- 本地最新修复已把视频抽帧改为“邻近时间点重试 + 异常帧跳过 + requestAnimationFrame 等待可渲染帧”，用于覆盖更多分辨率、时长边界和解码抖动场景
- 本地最新修复已把历史记录读取改为强校验，并把轨迹叠加画布缩放到安全尺寸，优先压低微信端在高分辨率视频上的结果渲染崩溃风险
- 本地最新修复已把电脑端结果区改成更均衡的双列模块化布局，重点收紧结果卡片尺寸和按钮断行问题
- 最新真实网址复测发现 `vision_wasm_internal.wasm` 直接走 `cdn.jsdelivr.net` 仍会超时，但 `fastly.jsdelivr.net`、`gcore.jsdelivr.net` 与 `unpkg` 在浏览器里可返回 200，当前已把远端热修复改为优先重写到 Fastly 节点
- 本轮继续处理的首要检查点是：线上 Pages 是否已经同步到 Fastly 运行时热修复，避免继续被旧静态包缓存误导
- 按全屏桌面视角继续审视后，本地页面又进一步收紧为更均衡的双列工作台，并缩小了未出结果时的空白占位高度
- 当前真实 Pages 已确认切到新版 `app/page.tsx`，但桌面端仍需继续压缩信息密度和按钮文案，避免用户体感上仍像“没改”
- 本地最新桌面端已进一步改成“窄控制栏 + 宽工作区”结构，并把按钮文案缩短为“最近记录 / 导入训练视频 / 微信快速上传 / 开始姿态分析”
- 当前已新增 `.eslintrc.json`，使 `npm run lint` 不再卡在首次交互式初始化
- 当前本地 `typecheck`、`lint`、`build` 已再次执行，正在优先执行“推送部署同步本地”，把最新布局和分析链路一起送上远端
- 产品侧鲁棒性增强已落地，后续仍可继续补更多动作专项规则与更细分的专业反馈模板

## 当前修复方向

- 使用用户提供的真实视频复现识别失败问题，优先提升分析成功率
- 固化 MediaPipe 运行时备用来源与识别降级策略，减少真实视频因运行时或有效帧不足直接失败
- 优化桌面端与手机端的布局、按钮与结果面板，让界面更接近专业视频分析软件
- 已完成本地 `typecheck`、`lint`、`build` 校验，下一步优先同步远端并继续用真实平台网址做桌面端/微信端全链路验证
- 等待 Pages 同步最新 Fastly CDN 运行时热修复后，继续直接用真实网址复测上传、分析、结果展示
- 继续补强后方与侧后方机位的专项分析深度
- 继续迭代更多动作的机位适配与专项技术规则
- 继续细化专业反馈模板，让结果更贴近真实教练复盘语言
- 继续优化手机端屏幕适配与微信浏览器体验
