# 项目框架

## 当前定位

- 项目名称：AI 运动教练
- 当前核心链路：上传训练视频 → MediaPipe 本地识别 → 轨迹与速度分析 → 输出动作建议 → GitHub Pages 部署
- 当前优先级：先保证线上 Pages 可访问，再继续细化动作识别和移动端体验

## 当前目录结构

- `app/`：Next.js App Router 页面与组件
- `app/components/`：视频轨迹、速度图表等核心前端组件
- `lib/`：动作分析类型与 MediaPipe 本地分析逻辑
- `.github/workflows/`：GitHub Pages 自动部署流程
- 根目录文档：记录实现状态、优化方案、测试说明与项目长期记录

## 当前核心实现

- 已接入 MediaPipe 本地姿态识别
- 已支持深蹲、卧推、高翻、硬拉与自动识别
- 已输出轨迹、速度、加速度、评分、风险与建议
- 已补充关键姿态指标与动作时间线复盘面板
- 已改为静态导出并通过 GitHub Actions 发布到 `gh-pages` 分支
- 当前进入识别鲁棒性增强阶段，重点提升多角度、多分辨率、低清视频下的兼容能力
- 已加入机位识别、清晰度/覆盖度评估、低质视频降级分析与更专业的训练建议输出
- 已进一步补充侧后方、后方机位识别与后链/下肢对线分析能力
- 已开始针对手机端小屏和微信场景优化上传与结果阅读体验

## 当前主要问题

- GitHub Pages 地址已经可以打开，但当前真实测试视频仍可能提示“无法分析，请更换更清晰的视频”
- 当前已定位到线上真实视频失败还包含 MediaPipe 运行时资源通过 CDN 加载时的 MIME 异常，正在切换为本地静态资源
- 电脑端界面整体布局与专业感不足，左右区域、按钮层级与信息密度仍需优化
- 当前需要先提升真实视频识别成功率与专业反馈质量，再继续把更完整的后方机位分析增强和手机端优化稳定同步到线上
- 产品侧鲁棒性增强已落地，后续仍可继续补更多动作专项规则与更细分的专业反馈模板

## 当前修复方向

- 使用用户提供的真实视频复现识别失败问题，优先提升分析成功率
- 固化 MediaPipe 本地运行时与识别降级策略，减少真实视频因运行时或有效帧不足直接失败
- 优化桌面端与手机端的布局、按钮与结果面板，让界面更接近专业视频分析软件
- 继续补强后方与侧后方机位的专项分析深度
- 继续迭代更多动作的机位适配与专项技术规则
- 继续细化专业反馈模板，让结果更贴近真实教练复盘语言
- 继续优化手机端屏幕适配与微信浏览器体验
